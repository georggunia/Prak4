Kompilieren Sie den Simulator mit

make

Um alle Konfigurationen mit allen bereitgestellten Cache-Traces zu
simulieren verwenden Sie

make simulateall

HINWEIS: Stellen Sie sicher, dass Sie die Konfigurationsdatei in die
Datei configurations.txt kopieren kopieren
(z.B. cp configurations4a.txt configurations.txt für Praktikum 4).

Lesen Sie auch den Hinweis zum Line-Spanning in
line_spanning_accesses.txt.

Jetzt sind Sie bereit, den CACHE SIMULATOR in CacheSimulator.java zu
implementieren.  Sie sollten nur diese Datei ändern müssen, aber Sie
können natürlich jede Datei hinzufügen oder jede andere Datei ändern,
wenn Sie möchten.

Sobald Sie Ihren Cache-Simulator implementiert haben und er korrekt
ist, sollte Ihre Ausgabe für die Konfigurationsdatei
configurations<ID>.txt mit der Ausgabe in output<ID>.txt
übereinstimmen z.B. für configurations4.txt sollten Sie output4a.txt
erhalten
